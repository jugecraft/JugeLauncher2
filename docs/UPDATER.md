# Auto-Updater

JugeLauncher uses the standardized **Tauri Updater** protocol.

## Workflow
1.  **Check**: On startup, the app queries `https://releases.jugelauncher.com/latest.json`.
2.  **Verify**: The response includes a signature generated by the private key. The app verifies this using the public key in `tauri.conf.json`.
3.  **Download**: If a new version exists, it downloads the bundle (NSIS for Windows, AppImage for Linux).
4.  **Install**: The installer is run automatically (passive mode) or replaced on restart.

## Configuration
Keys are generated using `tauri signer generate`.
- **Private Key**: Stored in CI/CD secrets (`TAURI_SIGNING_PRIVATE_KEY`).
- **Public Key**: Stored in `tauri.conf.json`.

## Release Channel
To push an update, tag a new release in GitHub. The CI workflow will build, sign, and upload the assets and the `latest.json` manifest.
